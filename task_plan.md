## 已做决策
| 决策 | 理由 |
|------|------|
| 在当前干净的 `main` 工作区原地执行 | 用户确认工作区已清理并要求继续前置安装 |
| 本地 `reference-design-contract` skill 生成 `DESIGN.md` 与参考页 | 纯本地文件工作流，符合已批准范围 |
| 使用 React reducer、原生 SVG、无头 Chrome 打印 | 一条命令生成交付 PDF，且不增加状态库、图表库、PDF 库 |
| 记录由本地 Node 服务原子写 JSON | 支持中途退出分析；静态部署可自然降级 |
| Logo 使用 SVG 而非 PNG | Logo 仅需清晰缩放的扁平标记，避免为单色图形增加光栅转换和依赖 |

## 遇到的错误
| 错误 | 尝试次数 | 解决方案 |
|------|---------|---------|
| 暂无影响开发的错误 | 0 | — |
| 验证脚本使用 zsh 保留变量 `path`，导致同一 shell 后续找不到命令 | 1 | 改用普通变量 `target` 后重跑 |
| 沙箱与浏览器插件运行时限制阻止标准本地预览路径 | 2 | 不修改插件；改用获批的 Chrome 无头渲染，并单独校验交互脚本 |
| 首次脚本语法检查未剥离 HTML 标签 | 1 | 按行删除 `<script>` 块首尾标签后重跑通过 |
