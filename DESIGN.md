# 咖啡冲煮入门 · Design Contract

目标表面：面向咖啡初学者和现场讲师的 16:9 互动教学课件；同一视觉系统同时服务浏览器放映与 HTML 打印 PDF。

证据边界：

| 来源 | 置信度 | Keep | Change | Do not copy |
|------|--------|------|--------|-------------|
| 已批准产品规格 | provided | 温暖编辑感、奶油底、咖啡棕、琥珀色、16:9 | 将抽象风格落实为可编码令牌 | 不增加未批准的品牌事实 |
| 用户提供的低对比度截图 | observed | 浅色卡片与信息分组 | 所有浅色表面显式使用深色文字 | 不延续近白标题、暗色后台面板或诊断页观感 |
| 咖啡冲煮主题 | inferred | 滤纸、粉水刻度、记录手记的材料语言 | 转译为弧线、刻度轨和编号 | 不使用写实咖啡店陈词、霓虹或黑金“高级感” |

## 1. Visual Theme & Atmosphere

唯一方向是“冲煮手记”：像一本摊开的咖啡实验记录册，温暖、安静、可操作。画面以大块留白和明确阅读顺序为主，只在一个位置使用由滤纸弧线和粉水刻度组成的主题标记。

- 情绪：清醒、温暖、可信，不卖弄专业术语。
- 材料：未漂白滤纸、陶瓷、铅笔刻度、咖啡液色。
- 识别元素：`brew rail`——页面边缘的一条 01–08 冲煮刻度轨；它表达真实课程进度，不作纯装饰。
- 视觉风险控制：琥珀色只做面积有限的强调，不承载白色小字。

## 2. Color

### Core tokens

运行时值由 `course.json` 注入；下表是当前示例课程值，不是引擎常量。`surface`、`muted`、`line` 缺失时只能从当前 JSON 核心颜色回退，图表色来自 `brand.chartColors`。

| Token | Value | Usage |
|-------|-------|-------|
| `--canvas` | `#F7F0E5` | 课件主背景 |
| `--surface` | `#FFF9F0` | 选项、详情和内容卡片 |
| `--ink` | `#2B211B` | 主正文；所有浅色容器必须显式声明 |
| `--muted` | `#6B5545` | 次要说明，禁止低于 16px |
| `--primary` | `#4A2C1A` | 标题、主要按钮、选中状态 |
| `--accent` | `#D89A4E` | 图表柱、进度和主题刻度 |
| `--bar-color` | `brand.chartColors[]` | 每个图表项的数据色 |
| `--line` | `#D8C7B5` | 1px 分隔线 |
| `--focus` | `brand.text` | 浅色表面上的 3px 键盘焦点环 |

### Contrast contract

- `--ink`、`--primary` 与 `--muted` 在 `--canvas` / `--surface` 上必须达到 WCAG AA 普通文本 `4.5:1`。
- `--accent` 只与 `--ink` 组合；不得在琥珀色上放白色普通文本。
- 主要按钮使用 `--primary` 背景和 `--surface` 前景；页眉、工具栏等深色表面使用 `--canvas` 文字。
- 图表不能只靠颜色：每根柱同时显示名称和数值，选中项再增加描边与详情面板。
- 不响应系统深色模式；课件前景色始终由这些令牌决定。

## 3. Typography

只使用系统字体，不请求远程字体。

- Display：`"Songti SC", "STSong", serif`，只用于 H1 和关键数字，制造纸本编辑感。
- Body：`"PingFang SC", "Microsoft YaHei", system-ui, sans-serif`，用于正文与控件。
- Utility：`"SFMono-Regular", Consolas, monospace`，用于页码、单位和短标签。
- H1：`clamp(56px, 6vw, 88px)` / `1.02`，字重 700，字距 `-0.035em`。
- H2：`clamp(36px, 4vw, 56px)` / `1.12`，字重 700。
- Body：`clamp(20px, 1.9vw, 28px)` / `1.55`。
- Label：16–18px / `1.4`，字重 700；投影场景不使用更小文字。
- 每行正文控制在 28 个中文字符以内，列表最多 4 条。

## 4. Spacing & Grid

- 基础单位：8px。
- 16:9 舞台内边距：桌面 56–72px；紧凑视口 28–36px。
- 主网格：12 列，列间距 24px；封面使用 7/5，图表使用 8/4，答题使用 5/7。
- 主要区块间距：32px；同组元素：12–20px。
- 卡片内边距：22–28px；圆角 14px，避免连续大圆角容器。
- `brew rail` 固定占 40px 宽，不覆盖正文区域。

## 5. Layout & Composition

### Cover

- 左侧标题和课程承诺占 7 列；右侧只放一个主题视觉，不堆叠信息卡。
- 标题下方用“风味 → 变量 → 记录”三段短语说明学习路径。
- 右缘 `brew rail` 显示当前页码，不硬编码课程总页数。

### Chart

- 标题位于左上；图表从统一基线向上生长。
- 详情面板占右侧 4 列，始终使用 `--surface` + `--ink`。
- 详情打开时不遮住柱形名称和数值。

### Quiz

- 左侧 5 列放问题、教学上下文与当前页码；右侧 7 列放选项、统计和“继续”。
- 主讲人重复加票或减票纠错时，在同一页面即时更新票数和百分比，不弹窗、不跳转。
- “继续”是结果区唯一主要操作。

### Responsive

- 宽度低于 800px 时取消固定 16:9，主网格改为单列，正文先于视觉。
- 图表详情移到柱形下方；答题选项保持至少 48px 点击高度。
- 打印始终恢复 16:9 横向，不使用移动布局。

## 6. Components

- `Stage`：奶油底、1px `--line`、18px 外圆角；打印时去掉外框和阴影。
- `BrewRail`：使用 JSON 主色显示当前页码。
- `PrimaryButton`：`--primary` 底、`--surface` 字、12px 圆角、14×22px 内边距。
- `SecondaryButton`：透明底、`--primary` 文字和边框。
- `ChoiceCard`：浅纸面、深色文字、2px 边框；唯一领先时使用咖啡棕底白字并显示 JSON 配置的领先标签。
- `BarButton`：琥珀/陶土/橄榄填充，名称和数值直接可见；键盘与鼠标行为一致。
- `DetailPanel`：浅纸面、深色正文、左侧 3px 主题色标记；关闭按钮有文字标签。
- `ResultRow`：选项名称、进度轨、票数与百分比四项同时出现。
- `CourseNav`：位于舞台外下方，仅含上一页、页码、下一页；总结页禁用下一页。
- 工具页组件不得出现在任何教学页面或打印页面。

## 7. Motion & Interaction

- 页面进入：整体 `opacity 0 → 1`、`translateY(8px) → 0`，180ms。
- 图表选择：只动画 `transform` 与 `opacity`，详情 180ms 淡入；柱高不重复弹跳。
- 答题：现场计数变化使用 140ms 状态过渡；不使用庆祝粒子或持续动画。
- 焦点：浅色表面使用 3px `--focus` 外环，深色页眉/工具栏使用 `--canvas` 外环，均保留 2px 间隔。
- `prefers-reduced-motion: reduce` 下取消所有非必要过渡。
- 零票或平票时不能越过答题页面；返回后允许修正票数并改走新的最高票分支。

## 8. Voice & Brand

- 语气像讲师现场指导：短句、主动语态、一次只解释一个动作。
- 用“记录这一杯”“查看明细”“继续”这类具体动词，不用“探索无限可能”等宣传句。
- 图表数字明确标单位，示例数据不暗示医疗建议或绝对结论。
- CTA 属于课程总结，只引导下一次冲煮练习；PDF、记录状态和校验不出现在 CTA。
- AI 生成和网页来源记录在 `sources.json`，不在课件正文堆叠出处。

## 9. Anti-patterns

- 禁止浅色卡片上的白色、米白色或低透明度标题。
- 禁止暗色仪表盘、后台管理侧栏、玻璃拟态和霓虹光晕。
- 禁止无意义的三等分卡片、胶囊标签墙和装饰性 KPI。
- 禁止远程字体、HTTP 图片和未经地址校验的远程素材；运行时图片只接受 HTTPS。
- 禁止把答题做成教学流程之外的弹窗或独立系统。
- 禁止将 PDF、记录状态、重新开始、配置校验放入第 8 页总结。
- 禁止只依赖颜色表达图表差异、选中状态或错误。
- 禁止小于 16px 的投影文字、超过 4 条的主列表和被裁切的长标题。
- 禁止为了动画引入运行库；CSS 已足够。

第一张实现页面必须证明：16:9 投影可读、浅色表面文字清晰、`brew rail` 与咖啡主题相关、键盘焦点可见，并且在 800px 以下能顺序阅读。
