# 主讲人现场计票设计

日期：2026-07-14

## 目标

选择题由主讲人根据台下听众回答现场计票。页面不再使用预置票数；主讲人可重复加票、纠正误点，最终由唯一最高票选项的 `goto` 决定后续分支。

## 交互

- 每个选项从 0 票开始；点击选项主体一次增加 1 票。
- 每个选项提供减 1 操作，票数最低为 0。
- 总人数、票数和百分比只根据本次现场计票计算，并随操作即时更新。
- 唯一最高票选项显示领先状态。
- 总票数为 0 或最高票平票时，“继续”不可用；主讲人补记一票后继续。
- 返回选择题时保留本次计票，主讲人可修正后改走新的最高票分支。

## 数据与分支

- 会话状态以“页面 ID → 选项 ID → 非负整数”保存票数。
- 加票和减票均由 reducer 处理；减票不得低于 0。
- `getNextSlideId` 只在存在唯一最高票时返回该选项的 `goto`，否则返回 `null`。
- 删除课程 JSON、类型、校验和文档中的 `initialVotes`，避免预置数据被误认为现场结果。
- 新增控件所需的可访问标签来自 JSON 的 `ui`，保持换课只改 JSON 的约束。

## 范围外

不增加听众扫码、多终端同步、服务端投票聚合或投票锁定阶段。当前场景只有主讲人操作课件，本地会话状态足够。

## 验证

- reducer 测试覆盖重复加票、减票下限、零票、平票、唯一最高票和返回后改票。
- 渲染测试确认页面从 0 开始，只显示现场计数，并使用 JSON 标签。
- 当前咖啡 JSON 与替换课程 JSON 均通过校验；全量测试和生产构建通过。
