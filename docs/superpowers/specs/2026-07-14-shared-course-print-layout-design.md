# 课程页与打印页共享画布设计

日期：2026-07-14

## 目标

`/course` 与 `/print` 对同一个页面定义使用同一套教学画布、内容结构和视觉样式。打印版不再重新拼装封面、正文、图表、答题和结束页；它只负责排列全部页面、生成图表明细续页并应用纸张分页规则。

## 一致性边界

主页面必须保持一致：

- 标题、正文、图片、图表、选项和 CTA 来自同一个 JSON 对象。
- 网格、字号、间距、颜色、图片位置、进度轨位置和主要视觉结构相同。
- 封面不在打印版额外插入网页中不存在的品牌信息行。

允许不同：

- `/course` 只显示当前路径中的一页；`/print` 按 JSON 顺序显示全部页面定义，包括互斥分支。
- `/course` 保留导航、计票、点击钻取和记录行为；打印版隐藏交互按钮。
- 打印页码反映导出顺序，不要求与某一次分支路径的访问序号相同。
- 答题页打印问题与选项，不打印或伪造尚未发生的现场票数。

## 图表与明细续页

每个图表先生成一个与 `/course` 初始状态相同的主页面：显示标题和完整图表，不在主页面强行塞入全部明细。

图表主页面之后自动生成明细续页：

- 续页编号使用主页面导出序号加字母，例如 `03A`、`03B`。
- 每个续页最多显示三个明细卡片，顺序与 `chart.series[]` 一致。
- 明细卡片继续读取 `details[series.detail]`，不维护第二份数据。
- 当前咖啡课程每个图表有三个明细，因此每个图表增加一个续页；九个主页面最终导出十一页。
- 不通过缩小正文、裁切、滚动区域或隐藏事实来维持固定页数。

若未来单个明细包含异常长文本，配置校验和内容审查负责控制可读长度；本次不引入浏览器测量分页算法。

## 组件边界

- 共享画布负责 16:9 容器、进度轨、页面标题区域和主题变量。
- 封面、正文、图表、答题和 CTA 的教学结构由课程页与打印页共同调用。
- 课程页在共享结构外增加导航和交互处理。
- 打印页在共享结构外增加工具栏、主页面循环、图表明细续页和 CSS 分页。
- 图表明细续页使用现有详情视觉令牌，但不属于互动教学路径。

## 数据与运行影响

- 不修改 `course.json` 字段结构、页面顺序或分支语义。
- 不修改 reducer、现场计票、记录 API 或服务端。
- `npm run export:pdf` 继续访问 `/print`，输出文件名保持 `exports/course.pdf`。
- 图片加载检查必须覆盖主页面和明细续页中的全部图片。

## 验证

- 渲染测试证明同一个封面定义在课程页和打印主页面复用同一教学结构。
- 渲染测试证明打印版包含九个主页面和两个图表明细续页。
- 打印版包含全部分支与全部明细，不包含工具页、网页导航按钮或现场假票数。
- 浏览器对照检查封面、正文、图表、答题和 CTA 五种页面。
- 重新执行配置校验、全量测试、生产构建和 `npm run export:pdf`，确认 PDF 为十一页且图片可离线查看。
