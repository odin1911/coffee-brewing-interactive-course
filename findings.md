# 发现与决策

## 需求
- 教学答题穿插在 8 页单分支学习路径中。
- `/course` 只放教学；PDF、记录状态、重开和校验位于独立 `/tools`。
- `/print` 输出 9 个页面定义、两条分支与所有图表明细。
- 本地运行持续写一个会话 JSON；每次重开新建文件；静态部署禁用记录。
- 所有课程内容来自 AI、用户提供文档或可审计网页来源。
- 设计准备只提供 `DESIGN.md` 和三张参考页面。

## 研究发现
- 当前仓库在 `main`，开始前 `git status --short` 无输出。
- `docs/skills/Contract.md` 当前为空，后续随实现逐步填写。
- 本地 `reference-design-contract` skill 已保留，可独立指导 Codex 写 `DESIGN.md`。
- 用户限定设计阶段只提交 `DESIGN.md` 和三张参考页，因此不生成该 skill 默认建议的额外交接文件。

## 技术决策
| 决策 | 理由 |
|------|------|
| 原地执行，不创建 worktree | 用户确认工作区已经干净 |
| 优先原生 CSS/SVG/打印能力 | Ponytail full；减少依赖和维护面 |
| TDD 仅覆盖有分支、循环、校验和持久化的逻辑 | 保留有效回归保护，不给静态文案制造测试噪音 |

## 遇到的问题
| 问题 | 解决方案 |
|------|---------|
| 额外设计工具不在项目范围内 | 删除相关程序与临时安装文件，仅保留本地 skill |
| zsh 的小写 `path` 是与 `PATH` 绑定的特殊数组 | shell 循环变量改用 `target`，避免覆盖命令搜索路径 |

## 视觉/浏览器发现
- 视觉方向确定为“冲煮手记”：暖纸张底色、深咖啡文字、琥珀/陶土/橄榄数据色，不使用默认 SaaS 卡片风格。
- 贯穿元素为右侧 `brew rail`（01–08 页码刻度），用于建立课件连续性；封面另用同心冲煮环作为识别图形。
- 关键前景/背景组合对比度最低为 5.21:1，正文与控件文字均达到 WCAG AA，修复了此前浅色卡片中文字不清的问题。
